## One-Line Pitch
The only FSM built for irrigation contractors' seasonal reality — spring startup batch-booking wizard, property zone documentation, and water restriction rules by ZIP code — replacing generic tools (Jobber, HousecallPro) that have no concept of seasonal surges.

## Problem

Irrigation contractors face three unique operational problems that generic FSM tools (Jobber, Housecall Pro) were never built for:

1. **Seasonal batch scheduling**: Spring startups and fall winterizations create massive demand spikes — a contractor needs to book 100–200+ jobs in a 3–4 week window, then go quiet for months. Generic FSM tools have no batch-booking workflow for this. Contractors are manually calling through their customer list or using spreadsheets during the surge window.

2. **Zone documentation per property**: Technicians arrive at properties without knowing zone counts, head types, valve locations, or controller model. System maps and zone notes need to be linked to each property and visible on the mobile app. Generic FSM doesn't support irrigation-specific property data (zone count, head type, controller model, valve box locations).

3. **Water restriction rules**: Many municipalities have odd-even watering schedules, drought restrictions, and specific watering windows that vary by HOA and municipality. Techs need to see the applicable local rule when scheduling a startup. No general-purpose FSM has a water restriction rule database by ZIP code.

## Product Concept: "SurgeFlow"

**Pricing**: $59/mo for up to 5 techs | Month-to-month
**Seasonal Pricing**: $49/spring + $49/fall = $98/year (vs $708/year at $59/mo) — available for seasonal operators
**LTD**: $79 (solo/2 tech), $129 (up to 5 techs)

**Core features**:

1. **Spring/Fall Surge Wizard** — select "Spring Startup" or "Fall Winterization," enter date range, pick neighborhoods → system auto-generates booking slots for all customers sorted by neighborhood/zone, filling the calendar in one pass. The feature no generic FSM has.

2. **Property Zone Documentation** — each customer property stores: zone count, head type per zone, controller model, valve box locations, last service notes, property photos. Technician sees all of this on the mobile app before arriving.

3. **Water Restriction Database** — ZIP-code lookup shows applicable watering restrictions (odd/even, drought alerts, HOA rules) when scheduling a startup. Auto-updated from municipal water authority data.

4. **Seasonal History** — one-tap comparison of spring 2025 vs spring 2026 for each property: what changed, what was repaired, what warnings were noted.

5. **Mobile Offline Mode** — zone notes and property maps available offline (essential for suburban backyards with poor cell signal)

6. **SMS batch-notify customers** — auto-text customers when their surge slot is booked: "Your spring startup is scheduled for April 12 between 8–10am."

**Key differentiators**:
- Seasonal batch-booking wizard = the single feature that makes every irrigation contractor's biggest operational bottleneck disappear
- Property zone documentation = eliminates the "I don't know this system" problem on every service call
- Seasonal pricing model = novel, addresses the off-season monthly-fee objection
- Water restriction database = local regulatory knowledge no generic FSM can replicate easily
