## One-Line Pitch
The first locksmith-only CRM: 24/7 dispatch, key-code database, property manager auto-invoicing, and emergency surge pricing — built for the 25K+ mom-and-pop locksmiths using generic tools that don't fit.

## Problem
Generic CRMs (ServiceTitan, Jobber) are designed for HVAC/plumbing and feel completely wrong for locksmiths. Locksmith workflows are unique: 24/7 dispatch with ETA texting, key-code lookup tables by vehicle/lock type, automotive lockout jobs, after-hours emergency pricing, property manager billing, and license/insurance document tracking. No indie SaaS player has built specifically for this. 25K+ locksmiths in North America alone are cobbling together generic tools, paper logs, and manual invoicing.

## Market Evidence
- 25K+ mom-and-pop locksmiths in North America
- 255+ tools listed on GetApp for locksmith scheduling — market is active but no dominant indie player
- Validated pricing: $49/user/month standard in the niche
- ALOA (Associated Locksmiths of America) has 9,000+ member companies — direct outreach channel
- r/Locksmith subreddit: 150K+ members

## Must-Have Filters
- [x] Problem is real (generic tools documented as wrong fit for locksmith workflows)

## Key Differentiators
1. Key-code lookup database (VIN → key cuts, lock → factory codes) — creates strong switching costs
2. 24/7 dispatch with automated SMS ETA to customers
3. Auto-invoice to property managers (high repeat billing use case)
4. After-hours emergency pricing rules (surge multiplier built in)
5. License and insurance document tracking (required for bonded operators)

## Next Steps
1. Validate key-code database sourcing options (ILCO, Ilias API, locksmith data providers)
2. Survey r/Locksmith on current software use and pain points (cold DM pilot users)
3. Check ALOA community forums for software discussion threads
4. Build MVP: dispatch board + SMS ETA + invoicing + key-code lookup
5. Launch cold-call pilot program via locksmith trade associations

## Risks
1. Smaller market (~25K) than FSM (~3.5M businesses) — limits MRR ceiling
2. Key-code database sourcing requires licensing from data providers
3. Locksmiths can be hard to reach digitally — trade association outreach required
