# Story 5.3: Customer CSV Import

Status: done

## Story

As a tradesperson switching from another tool,
I want to import my existing customer list from a CSV file,
so that I don't have to re-enter hundreds of customer records manually.

## Acceptance Criteria

1. **[Import Customers entry in Settings]** Given the user is on the Settings screen at `/(tabs)/more/`, when they scroll the list, then an "Import Customers" row is visible under a "Customers" section header, above the Calendar section. Tapping it navigates to `/(tabs)/more/import-customers`. (FR35)

2. **[CSV file selection via device picker]** Given the user is on the Import Customers screen, when they tap "Select CSV File", then `DocumentPicker.getDocumentAsync({ type: ['text/csv', 'text/plain', 'application/csv', '*/*'] })` is called and only CSV/text files are presented to the user. If the user cancels the picker, the screen returns to its initial state with no error shown.

3. **[Parse and preview first 5 rows with column mapping]** Given the user selects a valid CSV file, when parsing completes, then the screen shows: (a) a "Preview" section listing the first 5 data rows (header row excluded) with detected column mapping labels above each column, and (b) a column mapping UI where the user can confirm or adjust which CSV header maps to which customer field (name, phone, email, address_line1, city, state, zip). Auto-detection attempts case-insensitive header matching for common synonyms (e.g. "first name" + "last name" → `name`, "mobile" → `phone`, "address" → `address_line1`, "postal" → `zip`). At minimum `name` must be mappable for the import to proceed.

4. **[Import writes valid rows to WatermelonDB]** Given the column mapping is confirmed by the user, when they tap "Import X Customers", then for each valid row (row has a non-empty value in the mapped `name` column): a Customer record is created in WatermelonDB via `database.write()` with `accountId` set from the authenticated user's `user.accountId`, UUID generated by WatermelonDB, and all mapped fields populated. Unmapped or empty fields are stored as empty string `''`. All records created in a single `database.write()` batch for performance. (FR35)

5. **[Invalid rows skipped with summary]** Given the CSV contains rows where the mapped `name` column is empty or whitespace-only, when the import processes, then those rows are skipped (not written to WatermelonDB). After import completes, a summary is displayed: "Imported X customers. Skipped Y rows (missing name)." The X + Y count equals the total data row count (excluding header). (FR35)

6. **[Duplicate phone detection with skip/update choice]** Given the CSV contains a row whose mapped `phone` value matches the `phone` field of an existing WatermelonDB Customer record with the same `account_id`, when the import processes that row, then the user is asked (via `Alert.alert`) whether to Skip or Update the existing record. "Skip" leaves the existing record unchanged and increments the skipped count. "Update" overwrites the existing customer's fields with the CSV row values (name, email, address fields) within a `database.write()` call. Phone is never changed on update since it's the match key. Duplicate detection happens after valid-row filtering (AC #5) and before WatermelonDB writes. (FR35)

7. **[Progress and completion feedback]** Given the import is running, when rows are being written, then a progress indicator is shown (e.g., "Importing... X / Y"). When import completes, the screen shows the summary from AC #5, a "Done" button that navigates back to `/(tabs)/more/`, and a "Import Another File" button that resets the screen to initial state.

8. **[Large file handling — no UI freeze]** Given the CSV contains up to 500 rows, when the import runs, then the UI does not freeze. All parsing and WatermelonDB writes are performed inside an `async` function called with `await` (never blocking the React Native JS thread synchronously for more than one frame). A single `database.write()` batch handles all valid rows.

      type: ['text/csv', 'text/plain', 'application/csv', '*/*'],

    setMapping(detectColumns(parsed.meta.fields ?? []));

      return headers.find((h) =>

- [x] 2.5: Implement `handleImport()` — the core import function called when user taps "Import":

  ```typescript
  async function handleImport() {
    setPhase('importing');
    const db = database;
    const accountId = user?.accountId ?? '';
    let importedCount = 0;
    let skippedCount = 0;
    let updatedCount = 0;

    // Pre-fetch existing customers for duplicate detection (phone → Customer)
    const existingCustomers = await db.get<Customer>('customers')
      .query(Q.where('account_id', accountId))
      .fetch();
    const phoneMap = new Map<string, Customer>();
    for (const c of existingCustomers) {
      if (c.phone) phoneMap.set(c.phone.trim(), c);
    }

    const toCreate: Array<Record<string, string>> = [];
    const toUpdate: Array<{ existing: Customer; row: Record<string, string> }> = [];
    const skipped: Array<{ row: Record<string, string>; reason: string }> = [];

    // Classify rows
    for (const row of rows) {
      const nameVal = (row[mapping.name] ?? '').trim();
      if (!nameVal) {
        skipped.push({ row, reason: 'missing name' });
        continue;
      }
      const phoneVal = mapping.phone ? (row[mapping.phone] ?? '').trim() : '';
      const existing = phoneVal ? phoneMap.get(phoneVal) : undefined;
      if (existing) {
        // Will prompt user per-duplicate via Alert — collect for batch prompt
        toUpdate.push({ existing, row });
      } else {
        toCreate.push(row);
      }
    }

    // Handle duplicates sequentially with per-item Alert
    const confirmedUpdates: Array<{ existing: Customer; row: Record<string, string> }> = [];
    for (const dup of toUpdate) {
      const choice = await new Promise<'skip' | 'update'>((resolve) => {
        Alert.alert(
          'Duplicate Customer',
          `A customer with phone "${dup.existing.phone}" already exists (${dup.existing.name}). What would you like to do?`,
          [
            { text: 'Skip', onPress: () => resolve('skip') },
            { text: 'Update', onPress: () => resolve('update') },
          ],
          { cancelable: false },
        );
      });
      if (choice === 'update') {
        confirmedUpdates.push(dup);
      } else {
        skippedCount++;
      }
    }

    // Batch create all new customers
    if (toCreate.length > 0) {
      await db.write(async () => {
        for (const row of toCreate) {
          await db.get<Customer>('customers').create((record) => {
            record.accountId = accountId;
            record.name = (row[mapping.name] ?? '').trim();
            record.phone = mapping.phone ? (row[mapping.phone] ?? '').trim() : '';
            record.email = mapping.email ? (row[mapping.email] ?? '').trim() : '';
            record.addressLine1 = mapping.addressLine1 ? (row[mapping.addressLine1] ?? '').trim() : '';
            record.addressLine2 = mapping.addressLine2 ? (row[mapping.addressLine2] ?? '').trim() : '';
            record.city = mapping.city ? (row[mapping.city] ?? '').trim() : '';
            record.state = mapping.state ? (row[mapping.state] ?? '').trim() : '';
            record.zip = mapping.zip ? (row[mapping.zip] ?? '').trim() : '';
            record.notes = '';
            record.quickbooksCustomerId = '';
          });
          importedCount++;
        }
      });
    }

    // Batch update confirmed duplicates
    if (confirmedUpdates.length > 0) {
      await db.write(async () => {
        for (const { existing, row } of confirmedUpdates) {
          await existing.update((record) => {
            record.name = (row[mapping.name] ?? '').trim();
            record.email = mapping.email ? (row[mapping.email] ?? '').trim() : record.email;
            record.addressLine1 = mapping.addressLine1 ? (row[mapping.addressLine1] ?? '').trim() : record.addressLine1;
            record.addressLine2 = mapping.addressLine2 ? (row[mapping.addressLine2] ?? '').trim() : record.addressLine2;
            record.city = mapping.city ? (row[mapping.city] ?? '').trim() : record.city;
            record.state = mapping.state ? (row[mapping.state] ?? '').trim() : record.state;
            record.zip = mapping.zip ? (row[mapping.zip] ?? '').trim() : record.zip;
            // phone is NOT updated — it's the match key
          });
          updatedCount++;
        }
      });
    }

    skippedCount += skipped.length;
    setSummary({ imported: importedCount, skipped: skippedCount, updated: updatedCount });
    setPhase('done');
  }
  ```

  Import `Q` from `@nozbe/watermelondb` at top of file for the pre-fetch query.

### WatermelonDB Write Pattern

All writes follow the established pattern from `apps/mobile/app/(tabs)/customers/[id].tsx`:

```typescript
await database.write(async () => {
  await database.get<Customer>('customers').create((record) => {
    record.accountId = accountId;
    record.name = '...';
    // ...
  });
});
```

For batch imports, loop inside a **single** `database.write()` call — WatermelonDB transactions are atomic and performant when batched. Do **not** call `database.write()` once per row (500 separate transactions would be slow).

### Duplicate Detection — Pre-Fetch, Not Per-Row Lookup

Pre-fetch all existing customers for the account into a `Map<phone, Customer>` before the loop. This avoids 500 individual WatermelonDB queries for a 500-row import. The `Q.where('account_id', accountId)` query followed by a JavaScript Map construction is O(n) and completes in < 50ms for typical dataset sizes (NFR3 compatible).

```typescript
const existing = await db.get<Customer>('customers')
  .query(Q.where('account_id', accountId))
  .fetch();
const phoneMap = new Map(existing.filter(c => c.phone).map(c => [c.phone.trim(), c]));
```

Import `Q` from `@nozbe/watermelondb` at the top of `import-customers.tsx`.

### Phone Is Never Changed on Update

When the user chooses "Update" for a duplicate, the existing customer's `phone` field is **not** updated. Phone is the deduplication key — changing it would break the dedup logic on future imports. Only `name`, `email`, and address fields are overwritten.

### Alert.alert for Duplicate Prompts

React Native's `Alert.alert` is synchronous from a UI perspective but the callback is asynchronous. Wrap it in a `Promise` to await the user's choice:

```typescript
const choice = await new Promise<'skip' | 'update'>((resolve) => {
  Alert.alert('Duplicate Customer', '...', [
    { text: 'Skip', onPress: () => resolve('skip') },
    { text: 'Update', onPress: () => resolve('update') },
  ], { cancelable: false });
});
```

`cancelable: false` is important — prevents the alert from being dismissed without a choice, which would leave the promise hanging.

**Important**: If a CSV has many duplicates (10+), showing 10 sequential alerts is poor UX. For MVP this is acceptable per the AC as written, but note this as a known limitation in completion notes.

### `expo-document-picker` Usage Notes (SDK 52)

```typescript
import * as DocumentPicker from 'expo-document-picker';

const result = await DocumentPicker.getDocumentAsync({
  type: ['text/csv', 'text/plain', 'application/csv', '*/*'],
  copyToCacheDirectory: true,
});

// result.canceled is true if user dismissed the picker
if (result.canceled) return;

// result.assets[0].uri is the local file URI
const uri = result.assets[0]?.uri;
```

`copyToCacheDirectory: true` copies the file to a temporary cache location that `FileSystem.readAsStringAsync` can access. Without this, Android may return a content:// URI that FileSystem cannot read directly.

### PapaParse Header Mode

```typescript
import Papa from 'papaparse';

const result = Papa.parse<Record<string, string>>(csvString, {
  header: true,          // First row is treated as column names
  skipEmptyLines: true,  // Empty rows are ignored
});

// result.data: Array<Record<string, string>> — each row as { columnName: value }
// result.meta.fields: string[] — column names from header row
// result.errors: PapaParseError[] — any parse errors
```

When `header: true`, `result.data[0]` is the first data row (not the header). The header values become keys in each row object.
